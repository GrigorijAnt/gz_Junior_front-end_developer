Git, привет!

Git - это система контроля версий


Основные команды для адресной строки Git

Создание репозитория

cd url-адрес (Change Directory) - перейти в папку c проектом

cd ../ (Change Directory) - перейти папку уровнем выше

ls - информация о содержимом выбраной директории (папки с проектом)

git init (инициализация) - создать в папке проекта пустой локальный репозиторий в виде директории .git, в которой храниться история коммитов.

git add README.md - добавить файл с текстом описания проекта

atom . - открыть проект из командной строки в текстовам редакторе Atom


Добавление файлов в репозиторий

git status - просмотреть список новых или изменёных файлов. Не закомиченых

git add . - Индексировать (подготовить) все файлы в ожидание коммита

git rm --cached имяФайла - убрать фаил из ожидания коммита

git commit -m "комментарий изменений" - Зафиксировать (сохранить) индексированные файлы с комментарием в историю

git log - показать полную историю изменений (коммитов)

git log --oneline - показать краткую историю изменений (коммитов)


Отмена изменений. Откат к старым версиям

git checkout id-коммита - откат к определённому коммиту по id, для ПРОСМОТРА! изменений

git checkout имяВетки - возврат к последней версии комитов ветки и переход по веткам

git revert id-коммита - удаление сделанных изменений в файлах определённого коммита по id, но не сам коммит

:wq - выход из редактора vin (непонятка)

git reset id-коммита - удаление коммитов выше указанного id-коммита (откат до указанной версии id), но сделанные изменения в файлах остаются

git reset id-коммита --hard - удаление комитов выше указанного id полностью, откат до указанной версии всё

git push --force - отправка текущей версии локального репозитория на удалённый репозиторий


Работа с ветками

gir branch имяВетки - создание новой ветки

git checkout имяВетки - переход по веткам

git checkout -b имяВетки - создание новой ветки с переходом на неё

git branch -a, либо git branch - показать список всех локальных веток

git branch -av - показать список всех локальных и удалённых веток

git merge имяВетки - объединение веток, команда вводится при нахождение на той ветке, в которую вливается

git branch -d имяВетки - удаление ветки с указанным именем


Работа с удаленным репозиторием

git remote add origin https://github.com/GrigorijAnt/....git - подключение к удалённому репозиторию

git remote - проверка подключения к удалённому репозиторию (выводит: origin)

git push -u origin master ,либо git puch - перенос файлов на удалённый репозиторий с локального репозитория

git clone https://github.com/../...git - скачать (клонировать) файлы с удалённого репозитория. URL-адрес копируется с удалённого репозитория нажав кнопку Clone or download -> Clone with HTTPS

git pull - проверяет на удалённом репозитории изменения в файлах и добавляет (изменяет) изменения в локальный репозиторий
